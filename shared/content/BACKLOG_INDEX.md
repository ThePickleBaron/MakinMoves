## How to Create New Backlog Content

### Framework for Adding New Topics

1. **Identify the cycle:**
   - What repeats? (crypto booms, stock crashes, commodity cycles, commodity cycles, geopolitical disruptions)
   - What divergence forms? (market pricing vs. reality)
   - How long is the cycle? (timeframe to trade)

2. **Research the thesis:**
   - Find 3-5 credible data sources
   - Document the mechanism (why does this happen?)
   - Identify leading vs. lagging indicators
   - List the key metrics to track

3. **Create the template:**
   - Write 1-2 generic Twitter thread outlines
   - Make it adaptable (placeholders for prices/dates/examples)
   - Include 2-3 variations (bear angle, bull angle, neutral data angle)

4. **Add tracking sheet (if ongoing series):**
   - Weekly metrics to monitor
   - Data sources to check
   - Divergence scoring rubric
   - Content posting schedule

5. **Store in `shared/content/`:**
   - BACKLOG_[topic].md (generic templates)
   - SERIES_[topic]_PART[N].md (specific deep-dives)
   - SERIES_[topic]_TRACKER.md (if ongoing)

## How Orchestrator Fits In

1. **You update backlog content** (1-2x per week, 15-20 min total)
   - Customize with latest data
   - Add to `shared/content/` as new `.md` file
   - Update `CONTENT_QUEUE.md` to mark as "READY TO POST"

2. **Orchestrator posts automatically** (every 4 hours)
   - Reads CONTENT_QUEUE.md
   - Posts Twitter threads to your account
   - Logs metrics (engagement, impressions, etc.)
   - Updates `shared/metrics/DAILY_METRICS.md`

3. **You review metrics** (Friday evening, 5 min)
   - Which threads got best engagement?
   - Which topics resonate most?
   - Adjust next week's backlog accordingly
