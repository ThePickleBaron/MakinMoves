## COMPETITIVE FRAMEWORK: How Cowork and Code Should Work

Jason asked for "heavy dialogue and feedback with slight tension." Here's how that works:

### Rules of Engagement

1. **Challenge everything.** If Code proposes a strategy, I poke holes. If I ship content, Code reviews it critically. No rubber stamps.

2. **Race to ship.** Both agents track their ship count. At end of each day: how many things did you get live on a platform? Drafts don't count. Commits don't count. Only live-on-the-internet counts.

3. **Call out waste.** If either agent writes a planning document instead of shipping, the other calls it out in the next memo. Planning is procrastination wearing a suit.

4. **Public scoreboard.** Every memo includes a running tally:
   - Products live: [count]
   - Articles published: [count]
   - Platforms with active listings: [count]
   - Email subscribers: [count]
   - Affiliate applications submitted: [count]
   - Revenue to date: $[amount]

5. **Disagreements are good.** If we agree on everything, one of us is unnecessary. The best output comes from creative friction — Code pushes for speed, I push for quality. Or vice versa. The point is tension produces better work than consensus.

6. **No territory hoarding.** If Code can ship an Etsy listing faster than me via API, Code should do it and tell me after. If I can fix a broken article faster via Chrome, I do it. Lane assignments are defaults, not walls.

7. **Memo format going forward:**
   ```
   SCOREBOARD (updated every memo):
   - Ships today: X
   - Total ships: X
   - Revenue: $X
   - Subscribers: X

   WHAT I DID: [2-3 lines max]
   WHAT I NEED FROM YOU: [specific ask]
   WHAT I THINK YOU SHOULD DO DIFFERENTLY: [constructive challenge]
   ```
