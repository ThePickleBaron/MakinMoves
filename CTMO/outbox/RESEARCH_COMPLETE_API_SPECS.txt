KEY RESEARCH FINDINGS
================================================================================

GUMROAD
  Authentication: OAuth 2.0, Bearer token, non-expiring
  Key Endpoints: /v2/products, /v2/sales, /v2/licenses/verify
  Rate Limits: UNDOCUMENTED (estimated ~1-2 QPS)
  Polling: Every 5-10 minutes (144-288 req/day)
  Error Recovery: Exponential backoff on 429
  Production Ready: YES (but undocumented; requires monitoring)
  Recommendation: ✅ INTEGRATE

ETSY API v3
  Authentication: OAuth 2.0 PKCE, Bearer token
  Key Endpoints: /v3/listings, /v3/inventory, /v3/receipts, /v3/images
  Rate Limits: 10 QPS, 10,000 daily (sliding window)
  Polling: Every 5 minutes (288 req/day, safe margin)
  Error Recovery: Exponential backoff + jitter on 429
  Rate Limit Headers: X-RateLimit-Remaining, X-RateLimit-Reset
  Production Ready: YES (excellent docs, stable)
  Recommendation: ✅ INTEGRATE

WORDPRESS REST API
  Authentication: JWT token (plugin) or Application passwords
  Key Endpoints: /wp/v2/posts, /wp/v2/media, /jwt-auth/v1/token
  Rate Limits: None by default (server capacity limit)
  Polling: Not needed (push-based)
  Featured Image: 2-step process (upload media, link to post)
  Token Expiry: 10 minutes (default, configurable)
  Error Recovery: Refresh JWT token on 401, retry
  Production Ready: YES (stable, well-documented)
  Recommendation: ✅ INTEGRATE

MEDIUM API
  Authentication: OAuth 2.0 or self-issued tokens
  Key Endpoints: /v1/me, /v1/publications, /v1/posts
  Rate Limits: Undocumented
  Official Status: NO LONGER MAINTAINED by Medium
  User Restriction: Must be writer/editor in publication
  Production Ready: NO (official API deprecated)
  Recommendation: ❌ SKIP (optional manual fallback only)

REDBUBBLE
  API Status: NO OFFICIAL PUBLIC API
  Automation: Requires web scraping (Selenium/Puppeteer)
  Rate Limits: ~1 upload per 60 seconds (bot detection)
  Challenges: CAPTCHA blocking, UI changes, session management
  Maintenance: Very high (scripts break frequently)
  Production Ready: NO (fragile, high maintenance)
  Recommendation: ❌ SKIP (use third-party tool or manual uploads)

================================================================================
ERROR RECOVERY STRATEGIES
================================================================================

Gumroad (Undocumented):
  429: Exponential backoff (1s, 2s, 4s, 8s...)
  5xx: Retry 3 times with 5s backoff
  Timeout: Retry after 5s
  Logging: Track all failures for analysis

Etsy (Well-Documented):
  429: Exponential backoff + jitter (recommended)
  401: Re-authenticate (token expired)
  403: Check OAuth scopes
  5xx: Retry with exponential backoff
  Monitoring: Use X-RateLimit-Remaining header

WordPress (Token-Based):
  401: Refresh JWT token, retry
  403: Check API user permissions
  400: Validate request body (featured image upload often fails)
  Token Refresh: Automatic on expiration

Medium (Deprecated):
  403: User not writer/editor (expected error)
  Any error: Exponential backoff
  Fallback: Manual posting to Medium.com

Redbubble (Scraping):
  CAPTCHA: Manual intervention required
  Login Failed: Re-authenticate, retry
  Upload Failed: Validate image format/size
  Session Timeout: Re-login after 30 minutes
