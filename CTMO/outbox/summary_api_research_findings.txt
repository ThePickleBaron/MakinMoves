RATE LIMITS SUMMARY
================================================================================

Gumroad:
  - Undocumented (will discover via 429 errors)
  - Safe polling: Every 5-10 minutes
  - Daily cost: ~144-288 API requests
  - Retry: Exponential backoff on 429

Etsy:
  - 10 requests/second hard limit
  - 10,000 requests/day limit (sliding window)
  - Safe polling: Every 5 minutes (288 req/day)
  - Well within limits; monitor X-RateLimit-Remaining header

WordPress:
  - No built-in rate limiting
  - Push-based (no polling needed)
  - Can POST 1-2 articles per minute safely

Medium:
  - Undocumented, API no longer maintained
  - Skipping integration; optional manual fallback

Redbubble:
  - 1 upload per 60 seconds minimum (bot detection)
  - No API; requires Selenium automation
  - Fragile; skip unless COO specifically requests

================================================================================
AUTHENTICATION APPROACHES
================================================================================

Gumroad:
  Authorization: Bearer {ACCESS_TOKEN}
  Setup: OAuth 2.0 authorization code flow
  Complexity: Low
  Token expiry: Non-expiring (until revoked)

Etsy:
  Authorization: Bearer {ACCESS_TOKEN}
  Setup: OAuth 2.0 with PKCE (RFC 7636)
  Complexity: Medium (PKCE required)
  Token expiry: Varies (check response)

WordPress:
  Authorization: Bearer {JWT_TOKEN} or Basic auth
  Setup: Install JWT plugin + create API user
  Complexity: Low
  Token expiry: 10 minutes (default, configurable)

Medium:
  Authorization: Bearer {SELF_ISSUED_TOKEN}
  Setup: Create self-issued token (no OAuth needed)
  Complexity: Low
  Token expiry: Unknown (official API deprecated)

Redbubble:
  Authorization: Browser session (Puppeteer)
  Setup: Automate login + form filling
  Complexity: High
  Session expiry: 30 minutes

================================================================================
ERROR RECOVERY STRATEGIES
================================================================================

Gumroad (Undocumented limits):
  - 429 error: Exponential backoff (1s, 2s, 4s, 8s...)
  - 5xx error: Retry 3 times with 5s backoff
  - Timeout: Retry after 5s
  - Log all failures for analysis

Etsy (Well-documented):
  - 429 error: Exponential backoff + jitter (recommended)
  - 401 error: Re-authenticate (token expired)
  - 403 error: Check OAuth scopes
  - 5xx error: Retry with exponential backoff
  - Use X-RateLimit-Remaining header to predict limits

WordPress (Token-based):
  - 401 error: Refresh JWT token + retry
  - 403 error: Check API user permissions
  - 400 error: Validate request body
  - Handle featured image upload as 2-step process

Medium (Deprecated API):
  - 403 error: User not writer/editor in publication
  - All errors: Exponential backoff
  - Recommendation: Use caution, API may break

Redbubble (Scraping):
  - CAPTCHA detected: Manual intervention required
  - Login failed: Re-login + retry
  - Upload failed: Validate image format/size
  - Session timeout: Re-authenticate after 30 min
